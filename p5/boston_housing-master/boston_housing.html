将数据集分为训练数据集和测试数据集的好处：

  • 能充分利用已有数据对学习算法进行训练和验证；

编程练习3：定义衡量标准¶

如果不能对模型的训练和测试的表现进行量化地评估，我们就很难衡量模型的好坏。通常我们会定义一些衡量标准，这些标准可以通过对某些误差或者拟合程度的计算来得到。在这个项目中，你将通过运算决定系数 R2 来量化模型的表现。模型的决定系数是回归分析中十分常用的统计信息，经常被当作衡量模型预测能力好坏的标准。

R2的数值范围从0至1，表示目标变量的预测值和实际值之间的相关程度平方的百分比。一个模型的R2 值为0还不如直接用平均值来预测效果好；而一个R2 值为1的模型则可以对目标变量进行完美的预测。从0至1之间的数值，则表示该模型中目标变量中有百分之多少能够用特征来解释。模型也可能出现负值的R2，这种情况下模型所做预测有时会比直接计算目标变量的平均值差很多。

在下方代码的 performance_metric 函数中，你要实现：

  • 使用 sklearn.metrics 中的 r2_score 来计算 y_true 和 y_predict的R2值，作为对其表现的评判。
  • 将他们的表现评分储存到score变量中。

或

  • (可选) 不使用任何外部库，参考决定系数的定义进行计算，这也可以帮助你更好的理解决定系数在什么情况下等于0或等于1。

from sklearn.metrics import r2_score

def performance_metric(y_true, y_predict):
    """计算并返回预测值相比于预测值的分数"""
    
    score = r2_score(y_true,y_predict)

    return score

def performance_metric2(y_true, y_predict):
    """计算并返回预测值相比于预测值的分数"""
    
    y_mean = sum(y_true) / len(y_true)
    ss_tot = sum([(y - y_mean)**2 for y in y_true])
    ss_res = sum([(y - f)**2 for y, f in zip(y_true, y_predict)])
    score = 1 - ss_res / ss_tot

    return score

问题 7- 网格搜索（Grid Search）¶

什么是网格搜索法？如何用它来优化模型？

问题 7 - 回答:¶

同一模型采用不同参数效果不同。网格搜索法就是将模型中的多个参数的取值进行组合遍历，通过交叉验证确定最佳效果参数。它的好处是，只需增加几行代码，就能遍历多种组合。

它通过列举需要调整的参数和各参数的可能取值，尝试每一种取值组合带入模型函数进行验证，从而找到最佳的参数组合。

问题 8 - 回答：¶

  • K折交叉验证是将一训练集平均分成 K 个子集，每次取 1 个子集作为验证集，其他 K-1 个子集作为训练集，执行 K 次训练和验证，对 K 次训练结果进行平均。这样做的优点在于，所有的训练样本都被作为了训练和验证，每个样本都被验证了一次。

  • 用网格搜索法结合交叉验证来获取最佳参数，就是用每一种可能的参数值组合带入模型函数，对训练数据集进行交叉验证，每个参数组合得到一个训练结果，如果有 n 种参数组合则得到 n 个训练结果，得到最佳结果的参数组合即为最佳参数组合。

  • GridSearchCV中的'cv_results_'属性是字典类型，导入pandas的DataFrame中后，每一行表示用网格搜索的一种参数组合进行训练得到的相关结果，包括训练得分、测试得分、平均训练时间、平均测试时间、参数取值等。

  • 网格搜索如果不使用交叉验证，可以将所有训练数据用于训练而不要验证，也可以将训练数据再划分为训练集和验证集。相比之下，后者显然模型泛化能力更好。如果将训练数据固定划分为训练集和验证集，所有参数组合在训练集上训练，在验证集上跑分验证，这样选出的最佳参数组合是在该验证集上表现最好的参数，而如果该验证集不具代表性，就会影响模型在未知数据上的表现。使用交叉验证，能够尽量避免因样本划分不合理而导致最优参数选择错误的问题。

def fit_model(X, y):
    """ 基于输入数据 [X,y]，利于网格搜索找到最优的决策树模型"""
    
    cross_validator = KFold(3)
    
    regressor = DecisionTreeRegressor()

    params = {'max_depth':np.arange(1,11)}

    scoring_fnc = make_scorer(performance_metric)

    grid = GridSearchCV(regressor, params, scoring=scoring_fnc, cv=cross_validator)

    # 基于输入数据 [X,y]，进行网格搜索
    grid = grid.fit(X, y)

    return grid.best_estimator_

def cross_val_score(estimator, X, y, scoring = performance_metric, cv=3):
    """ 返回每组交叉验证的模型分数的数组 """
    #非乱序条件下平均分割数据集
    bounds = [(i * len(X) / cv, (i+1) * len(X) / cv) for i in range(cv)]
    
    scores = []
    for a,b in bounds:
        X_test = X.iloc[a:b]
        y_test = y.iloc[a:b]
        X_train = X.iloc[[i for i in range(len(X)) if i not in range(a,b)]]
        y_train = y.iloc[[i for i in range(len(X)) if i not in range(a,b)]]
        
        estimator.fit(X_train, y_train)
        y_predict = estimator.predict(X_test)
        scores.append(scoring(y_test, y_predict))

    return scores

模型健壮性¶

一个最优的模型不一定是一个健壮模型。有的时候模型会过于复杂或者过于简单，以致于难以泛化新增添的数据；有的时候模型采用的学习算法并不适用于特定的数据结构；有的时候样本本身可能有太多噪点或样本过少，使得模型无法准确地预测目标变量。这些情况下我们会说模型是欠拟合的。
